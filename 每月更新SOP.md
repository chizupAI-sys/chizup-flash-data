# ChizUP! 快閃據點每月更新 SOP

> 適用對象：非技術背景的店務人員
> 更新時間：每月快閃時間表確認後即可操作，整個流程約 3 分鐘

---

## 一、什麼時候需要更新？

每個月的快閃時間表確認後（通常是月初），就可以進行更新。
更新完成後，**所有商品頁的快閃資訊會在約 1 分鐘內自動同步**，不需要再去改每一個商品頁。

---

## 二、如何進入 GitHub 網頁編輯 JSON

### 步驟 1：開啟 GitHub 頁面

用瀏覽器打開以下網址（請先確認你有登入 GitHub 帳號）：

```
https://github.com/[你的GitHub帳號]/chizup-flash-data
```

你會看到一個檔案列表，裡面有 `flash-stores.json` 這個檔案。

---

### 步驟 2：點開 flash-stores.json

點一下 `flash-stores.json` 這個檔案名稱，進入檔案預覽頁面。

---

### 步驟 3：點鉛筆圖示開始編輯

在頁面右上角，會看到一個 **鉛筆（✏️）圖示**，點它就可以進入編輯模式。

> 如果你看到的是「Fork this repository」按鈕，代表你沒有編輯權限，請確認你有登入正確的 GitHub 帳號。

---

### 步驟 4：修改內容（詳見下一節）

---

### 步驟 5：儲存（Commit）

修改完之後，往頁面下方捲，會看到一個綠色的 **「Commit changes」** 按鈕。

- `Commit message`（說明欄）可以填：`update: 更新 XX 月快閃據點`
- 直接點 **「Commit changes」** 確認儲存

---

## 三、JSON 格式說明

### 完整格式長這樣：

```json
{
  "updated": "2026-05",
  "stores": [
    {
      "name": "台北美麗華",
      "floor": "B1",
      "start": "05/01",
      "end": "05/31"
    },
    {
      "name": "桃園台茂購物中心",
      "floor": "5F",
      "start": "05/01",
      "end": "05/31"
    }
  ]
}
```

### 欄位說明

| 欄位 | 說明 | 填寫格式 | 範例 |
|------|------|----------|------|
| `updated` | 這份資料的月份（僅供參考） | `"YYYY-MM"` | `"2026-05"` |
| `name` | 百貨名稱 | 文字 | `"台北美麗華"` |
| `floor` | 樓層 | 文字 | `"B1"` 或 `"5F"` |
| `start` | 快閃開始日期 | `"MM/DD"` | `"05/01"` |
| `end` | 快閃結束日期 | `"MM/DD"` | `"05/31"` |

---

### 新增一筆據點

在 `stores` 陣列的最後一筆資料後面，加一個逗號，然後加一組 `{ }` 資料：

```json
"stores": [
  {
    "name": "台北美麗華",
    "floor": "B1",
    "start": "05/01",
    "end": "05/31"
  },
  {
    "name": "新光三越信義",
    "floor": "7F",
    "start": "05/15",
    "end": "05/21"
  }
]
```

> ⚠️ 注意：最後一筆資料的 `}` 後面**不加逗號**，其他的都要加！

---

### 刪除一筆據點

把要刪除的那一組 `{ ... }` 整個刪掉，並確認前一筆的結尾沒有多餘的逗號。

---

### 修改一筆據點

直接在對應欄位改數字或文字就好。例如把 `"end": "04/30"` 改成 `"end": "05/15"`。

---

## 四、如何確認更新成功

儲存後，請在瀏覽器開啟以下網址（這是 JSON 的公開網址）：

```
https://raw.githubusercontent.com/[你的GitHub帳號]/chizup-flash-data/main/flash-stores.json
```

- 如果看到你剛才改的內容 → **更新成功！**
- 如果看到的還是舊的內容 → 等 30 秒後重新整理再看一次（GitHub 有快取，通常 1 分鐘內更新）

## 五、常見錯誤排查

### 問題 1：商品頁顯示「快閃資訊載入中，請參考官網最新消息」

這代表 widget 抓不到資料，可能原因：

| 原因 | 解法 |
|------|------|
| JSON 格式有錯（多了逗號、少了引號等） | 回 GitHub 重新檢查格式，用下方工具驗證 |
| GitHub raw 網址貼錯 | 確認 widget HTML 裡的 `JSON_URL` 是正確網址 |
| 網路暫時問題 | 等幾分鐘後重新整理商品頁 |

---

### 問題 2：如何檢查 JSON 格式有沒有錯？

把整份 JSON 內容複製起來，貼到這個網站檢查：

```
https://jsonlint.com
```

- 按 **「Validate JSON」**
- 看到綠色 **「Valid JSON」** → 格式正確
- 看到紅色錯誤訊息 → 依提示找到問題的那一行修正

---

### 問題 3：改了但商品頁還是沒更新

1. 先確認 raw 網址的內容已經是新的（參考第四節）
2. 清除瀏覽器快取後再開商品頁（按 Ctrl+Shift+R 或 Cmd+Shift+R）
3. 如果以上都試過還是沒更新，請聯絡網站管理員
